Mise en place d'un Self-Hosted Runner & Déploiement Web

Rapport de Travaux Pratiques - CI/CD GitHub Actions

DevOps Debian 13 GitHub Actions

1. Objectif du Projet

L'objectif de ce TP était de configurer un serveur Linux (Debian 13) pour agir en tant que 'Self-Hosted Runner' GitHub. Un runner est une machine dédiée à l'exécution de workflows CI/CD.

Contrairement aux runners hébergés par GitHub, un runner auto-hébergé permet un contrôle total sur l'environnement d'exécution (outils, accès réseau, OS). Le but final était de déployer automatiquement une page web via Nginx à chaque modification du code source sur la branche master.

2. Étapes de Réalisation

1

Configuration du Serveur

Installation de l'agent GitHub Runner sur la VM Debian 13, authentification avec un token d'accès, et configuration en tant que service système pour une exécution continue.

2

Tests de Communication

Création d'un premier workflow d'intégration continue (CI) pour valider que les commandes envoyées par GitHub s'exécutaient bien sur le serveur local.

3

Validation du Code

Mise en place d'une étape de vérification automatique de la syntaxe HTML à l'aide de l'outil natif tidy avant tout déploiement.

4

Déploiement Automatisé (CD)

Automatisation de l'installation de Nginx, configuration du Virtual Host et copie des fichiers de l'application vers le répertoire web /var/www/site.

3. Fichiers & Workflows

index.html (La page web) 

<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>Lab test</title>
</head>
<body>
<h1>Mise en ligne</h1>
<p>Site de demo par CI/CD</p>
</body>
</html>

test.yml (Test du Runner) 

name: Test Runner
on:
  push:
    branches:
      - master
jobs:
  test-runner:
    runs-on: [self-hosted, linux, x64]
    steps:
      - name: Verification
        run: echo "Test de fonctionnement du Runner !"
      - name: tests de commandes
        run: |
          echo "Runner OK"
          hostname
          whoami
          uname -a

verif.yml (Vérification CI) 

name: CI - Vérification HTML
on:
  push:
    branches: [ master ]
jobs:
  test-html:
    runs-on: [self-hosted, linux, x64] 
    steps:
      - name: Récupération du code
        uses: actions/checkout@v4
      - name: Installation du validateur HTML (Tidy)
        run: sudo apt-get update && sudo apt-get install -y tidy
      - name: Vérification syntaxe HTML
        run: tidy -q -e index.html

publish.yml (Déploiement Nginx) 

name: Installation et Publication de l'application
on:
  push:
    branches: [ master ]
jobs:
  deploy:
    runs-on: self-hosted
    steps:
      - name: Récupération du code
        uses: actions/checkout@v4
      - name: Installation nginx
        run: |
          sudo apt update
          sudo apt install -y nginx
          sudo mkdir -p /var/www/site
          sudo chown -R www-data:www-data /var/www/site
      - name: Configuration nginx
        run: |
          sudo tee /etc/nginx/sites-available/site > /dev/null <

4. Résultats Obtenus

La mise en place de la chaîne d'intégration et de déploiement continu est opérationnelle. Le code poussé sur le dépôt est intercepté, analysé, et déployé sur le serveur Nginx local automatiquement en quelques secondes.
